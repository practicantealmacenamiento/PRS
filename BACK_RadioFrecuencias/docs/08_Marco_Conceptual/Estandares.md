# Marco Conceptual

## Estandares de referencia

### ISO/IEC 25000 (SQuaRE)
Marco para evaluar la calidad del software considerando funcionalidad, rendimiento, usabilidad, confiabilidad, seguridad, mantenibilidad y portabilidad. Sirve como base para revisar los requisitos funcionales y no funcionales documentados en este repositorio y monitorear su cumplimiento durante cada release.

### IEEE 830 e IEEE 1063
- **IEEE 830**: guia estructurada para especificaciones de requisitos. Inspira el formato de los documentos de requisitos funcionales y no funcionales.
- **IEEE 1063**: establece buenas practicas para manuales y guias tecnicas. Se utiliza para estructurar esta documentacion, asegurando claridad y trazabilidad.

### ISO/IEC 12207
Define procesos del ciclo de vida del software (desarrollo, integracion, pruebas, mantenimiento). Orienta la estrategia de soporte y mantenimiento, garantizando evidencia documental de cada fase.

### ISO 9001
Enfoque de gestion de calidad y mejora continua. Se adopta para estandarizar registros de incidencias, cambios y auditorias, alineando el proyecto con politicas corporativas.

### Guia Scrum
Marco agil utilizado para planificar iteraciones, retrospectivas y revisiones de producto. Permite adaptar rapidamente reglas de negocio del backend segun feedback operativo.

### ISO/IEC 26550
Buenas practicas para gestion de configuracion y versionamiento. Respaldan el uso de Git, la generacion de tags semanticos y la trazabilidad en `audit_log`.

La combinacion de estos estandares proporciona un marco consistente para mantener la calidad tecnica, la gobernanza y la documentacion del backend RadioFrecuencias.
